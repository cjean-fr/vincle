# Components

Three primitives split the concern between declaring a placeholder and pushing
deferred content:

- **`<Slot>`** — a named hole with fallback content rendered immediately in the
  shell
- **`<Fill>`** — pushes deferred content into a slot, overriding its fallback
- **`<Defer>`** — a deferred region with a placeholder and fallback in one
  component
- **`<ClientFetch>`** — client-side fetch placeholder.

## `<Slot>` — named insertion point

Declares a named hole in the shell. Its `children` are rendered **immediately**
as placeholder content — visible until a `<Fill>` replaces them. An empty slot
renders an empty placeholder.

```tsx
import { Slot } from "@vincle/flow";

<Slot name="comments">
  <p>Loading comments…</p>
</Slot>;
```

A `Slot` registers nothing in the pending store — it is purely a passive hole.

`Slot` is the recommended way to define the layout's insertion points. The
`name` is **required** and must be a valid fragment id
(`/^[a-zA-Z][a-zA-Z0-9_-]*$/`).

## `<Fill>` — push content

Pushes deferred content into an existing slot by name. Renders nothing itself
(returns `null`).

`children` is deferred automatically — plain JSX works:

```tsx
<Fill target="page">
  <h1>Hello</h1>
</Fill>
```

### Merge types

`merge` controls how the content applies to the target element:

| `merge`     | Effect                                    |
| ----------- | ----------------------------------------- |
| `"replace"` | Target element is replaced (default)      |
| `"append"`  | Content inserted as last child of target  |
| `"prepend"` | Content inserted as first child of target |
| `"before"`  | Content inserted as previous sibling      |
| `"after"`   | Content inserted as next sibling          |

Not every adapter supports all merge types — an unsupported merge is rejected
at registration with a clear error.

## `<Defer>` — deferred region with placeholder

Declares a deferred region with a placeholder in the shell. The `fallback` prop
defines the content shown immediately. `children` is deferred automatically —
plain JSX works without a thunk:

```tsx
import { Defer } from "@vincle/flow";

<Defer name="dashboard" fallback={<p>Loading dashboard…</p>}>
  <HeavyDashboard />
</Defer>;
```

Pass a factory when you need the `AbortSignal` for cancellation:

```tsx
<Defer name="dashboard" fallback={<p>Loading dashboard…</p>}>
  {(signal) => <HeavyDashboard signal={signal} />}
</Defer>
```

`<Defer>` is equivalent to a `<Slot>` + `<Fill>` pair, but in a single
declaration. It is useful when a component needs its own placeholder without
relying on an external layout slot.

## Content forms

Both `<Fill>` and `<Defer>` accept children in either form:

| Children                     | Behaviour                                                          |
| ---------------------------- | ------------------------------------------------------------------ |
| Plain JSX                    | Deferred automatically — executes when the fragment renders        |
| `(signal: AbortSignal) => …` | **Cancellable** — receives `AbortSignal` (request + timeout abort) |

### Plain JSX (recommended for most cases)

`@vincle/core` renders JSX eagerly to HTML strings, but `@vincle/flow` captures
the output and defers it to the fragment's render slot. A child like
`<HeavyComponent />` is evaluated immediately, but its content is not sent to
the response until the fragment flushes. No thunk needed.

### Factory form — for cancellation

The factory function receives an `AbortSignal` tied to:

- The **request lifetime** — if the client disconnects, the signal aborts and
  any pending work can stop
- A **per-fragment timeout** — if the factory takes longer than `timeout` ms,
  the signal aborts with a timeout error

### One-shot vs stream

The return type of a factory (or the rendered result of plain JSX) determines
the behaviour:

| Return type            | Behaviour                                                |
| ---------------------- | -------------------------------------------------------- |
| A renderable JSX node  | One-shot patch — renders once and resolves               |
| `Promise<VNode>`       | Resolves asynchronously, then patches in                 |
| `AsyncIterable<VNode>` | A **stream** — one patch per item, as each item resolves |

The runtime unwraps `Promise` and `AsyncIterable` automatically inside the
factory — you never need to call `.then()` or `for await` yourself.

## Common props

| Prop       | Applies to      | Meaning                                                   |
| ---------- | --------------- | --------------------------------------------------------- |
| `name`     | `Slot`          | id of the placeholder (required)                          |
| `name`     | `Defer`         | id of the placeholder (auto-generated if omitted)         |
| `target`   | `Fill`          | target slot id to push content into                       |
| `fallback` | `Defer`         | placeholder content shown in the shell                    |
| `merge`    | `Defer`, `Fill` | how content applies to its target (default `"replace"`)   |
| `timeout`  | `Defer`, `Fill` | per-fragment render timeout in ms                         |
| `onError`  | `Defer`, `Fill` | per-fragment error handler, overriding the renderer's one |
