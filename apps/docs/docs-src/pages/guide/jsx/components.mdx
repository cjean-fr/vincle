# Components

Components are plain functions that return JSX. No classes, no hooks, no lifecycle — just props in, string out.

## Basics

A component receives props as its first argument and returns JSX:

## Typed props

Props are typed with standard TypeScript interfaces:

```tsx
interface CardProps {
  title: string;
  children: VincleNode;
}

function Card({ title, children }: CardProps) {
  return (
    <div class="card">
      <h2>{title}</h2>
      {children}
    </div>
  );
}
```

## Async components

Any component can be `async`. The whole tree is awaited by `renderToString`:

```tsx
async function Profile({ id }: { id: string }) {
  const user = await db.users.find(id);
  return (
    <div class="profile">
      <h1>{user.name}</h1>
      <p>{user.bio}</p>
    </div>
  );
}
```

Sibling async branches run concurrently — `renderToString` resolves only when every branch finishes.

## What you don't need

- **State** — there is no re-rendering. State lives outside the render (database, request context).
- **Effects** — no client lifecycle. Render once, produce a string, done.
- **Context providers** — Vincle uses scoped context via `withScope`, not wrapper components.
- **`React.memo`** — every render is a fresh call. Memoization is meaningless.
