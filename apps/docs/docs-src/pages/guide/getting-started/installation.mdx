# Installation

`@vincle/core` has zero runtime dependencies. Install it and configure
TypeScript — that's all.

## TypeScript configuration

Set `jsxImportSource` in your `tsconfig.json` so that JSX files don't need any
explicit import:

## Optional: @types/react

Installing `@types/react` gives you per-element attribute autocomplete (e.g.
`src` on `<img>`, `href` on `<a>`). It is not required — `@vincle/core` works
without it.

<CodeExample src="guide/getting-started/tsconfig-with-react.json" />

## What you get

| Export                  | Purpose                                    |
| ----------------------- | ------------------------------------------ |
| `renderToString`        | Renders a JSX tree to an HTML string       |
| `raw`                   | Marks a trusted HTML string (no escaping)  |
| `Fragment`              | Standard JSX fragment (`<>…</>`)           |
| `context` / `withScope` | Scoped context for per-request isolation   |
| `jsx-runtime`           | The JSX transform entry point (auto-wired) |

Everything else is internal. There is no plugin registry, no provider tree, no
config object to maintain.
