# raw()

Marks an HTML string as **trusted** — it is rendered verbatim without escaping.

## Signature

```tsx
raw(value: string): RawString
```

## When to use it

`raw()` is the escape hatch for trusted HTML sources — content that has already
been sanitised or generated by a trusted process:

```tsx
import { raw } from "@vincle/core";

// Markdown renderer output — trusted source
const html = await renderMarkdown(post.body);
<div>{raw(html)}</div>;

// DOMPurify-sanitised user content — explicitly cleaned
const safe = DOMPurify.sanitize(userInput);
<div>{raw(safe)}</div>;

// Pre-escaped template output — you know it's safe
<div>{raw("<strong>Trusted</strong>")}</div>;
```

## When NOT to use it

`raw()` bypasses all HTML escaping. Never use it with unsanitised user input:

```tsx
// ❌ XSS if userInput contains <script>alert(1)</script>
<div>{raw(userInput)}</div>;

// ✅ Safe — Vincle escapes by default
<div>{userInput}</div>;
```

If you need to render untrusted HTML as HTML (e.g. forum posts with basic
formatting), use a sanitizer like [DOMPurify](https://github.com/cure53/DOMPurify)
first, then pass the result to `raw()`.

## Alternative: dangerouslySetInnerHTML

Vincle also supports `dangerouslySetInnerHTML` for React compatibility:

```tsx
<div dangerouslySetInnerHTML={{ __html: "<b>bold</b>" }} />
```

This is equivalent to `raw()`. Same warning applies: never use with
untrusted content.
