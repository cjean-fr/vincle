# JSX types

The type system that powers Vincle â€” `VNode`, `Component`, and the `JSX`
namespace.

## VNode

The union of everything Vincle can render. Use it to annotate props that hold
markup, a component's `children`, or the return type of a helper that produces
content.

<CodeExample src="api/renderToString/vincle-node.ts" />

`renderToString` accepts exactly one `VNode`. To render multiple siblings,
wrap them in a fragment (`<></>`).

Prefer `VNode` over `JSX.Element` in your own annotations:
`JSX.Element` is the factory's return contract and resolves to whichever global
`JSX` namespace is active â€” typically React's when `@types/react` is present.
`VNode` is a plain export and can never be captured by that global.

### What each branch means

| Type                             | Example                            | Behaviour                                                   |
| -------------------------------- | ---------------------------------- | ----------------------------------------------------------- |
| `string` / `number` / `bigint`   | `"hello"`, `42`, `100n`           | HTML-escaped and rendered as text                           |
| `boolean` / `null` / `undefined` | `true`, `null`                     | Renders nothing (empty string)                              |
| `RawString`                      | `raw("<b>bold</b>")`              | Passes through verbatim, no escaping                        |
| `Promise<VNode>`            | `Promise.resolve(<p>hi</p>)`       | Awaited automatically by `renderToString`                   |
| `VNode[]`                   | `[<a/>, <b/>]`                     | Flattened and rendered in order                             |
| `Iterable<VNode>`           | `Generator`, `Set`, `Map.values()` | Iterated and rendered lazily                                |
| `AsyncIterable<VNode>`      | `AsyncGenerator`                   | Iterated and awaited per-item, streaming                    |

## Component

The type of a Vincle component â€” a function that receives props and returns a
`VNode`.

```tsx
import type { Component } from "@vincle/core";

interface GreetingProps {
  name: string;
}

const Greeting: Component<GreetingProps> = ({ name }) => {
  return <h1>Hello, {name}!</h1>;
};
```

Components can be `async` â€” `renderToString` awaits them automatically:

```tsx
const AsyncGreeting: Component<{ id: string }> = async ({ id }) => {
  const user = await db.users.find(id);
  return <h1>Hello {user.name}</h1>;
};
```

The `Component<P>` type adds `children?: VNode` and common HTML
attributes (`class`, `style`, `id`, etc.) to your props, so you don't need
to declare them yourself.

## JSX namespace

Vincle declares its own `JSX` namespace so the TypeScript compiler knows how
to type-check JSX expressions.

| Export                    | Purpose                                                                    |
| ------------------------- | -------------------------------------------------------------------------- |
| `JSX.Element`             | Alias for `VNode` â€” the union of everything renderable                     |
| `JSX.ElementType`         | What can appear as a tag â€” a string (HTML element) or a component function |
| `JSX.IntrinsicElements`   | Maps lowercase tag names to their attribute types                          |
| `JSX.IntrinsicAttributes` | Additional attributes on every element (`key`)                             |

You rarely need these directly â€” they're wired automatically by the `jsxImportSource`
in your `tsconfig.json`. They become relevant when you write generic utilities
that operate on JSX nodes or component factories.

## StaticAttributes\<T\>

A generic utility type that strips runtime-only properties from an attribute
shape. Given a type `T`, it removes event handlers (`on*`), `children`, `style`,
`class`, and `className` â€” the props that are consumed by the JSX runtime rather
than serialized as HTML attributes â€” and re-adds them with Vincle-compatible
types (`Awaitable`, `CSSProperties`).

```tsx
import type { StaticAttributes } from "@vincle/core";

interface MyProps {
  href: string;
  onClick: () => void; // will be stripped by StaticAttributes
  rel: string;
}
// StaticAttributes<MyProps> = { href: string; rel: string }
//   & { class?: ...; style?: ...; children?: ...; dangerouslySetInnerHTML?: ... }
//   & StringEventHandlers
```

Use it when writing a generic wrapper that projects custom props onto a
native element:

```tsx
function SafeLink<T extends {}>(props: StaticAttributes<T> & { to: string }) {
  return <a href={props.to} {...props} />;
}
```

`StaticAttributes` is the building block of `HTMLAttributes` â€” you rarely need
it directly, but it's exported for library authors who define custom element
types.

## HTMLAttributes

The base attribute type shared by all HTML elements. It includes:

- **Global attributes** â€” `id`, `class`, `style`, `lang`, `dir`, `role`, `hidden`, `title`, `tabIndex`
- **Event handlers as strings** â€” `onClick`, `onSubmit`, `onChange`, etc., all typed as `string` (not functions)
- **`children`** â€” typed as `VNode`
- **`dangerouslySetInnerHTML`** â€” React-compatible API for raw HTML
- **Catch-all** â€” `[key: string]: any` allows any valid HTML attribute or `data-*` attribute

```tsx
import type { HTMLAttributes } from "@vincle/core";
```

When `@types/react` is installed, `HTMLAttributes` merges with
`React.HTMLAttributes` to provide per-element autocomplete (e.g. `src` on
`<img>`, `href` on `<a>`). This is optional â€” Vincle works without it.

## CSSProperties

Style objects use the same camelCase properties as React:

```tsx
const styles = {
  backgroundColor: "#fff",
  fontSize: "14px",
  "--custom-var": "value", // CSS variables supported
} satisfies CSSProperties;
```

`CSSProperties` allows any key â€” useful for CSS variables and custom
properties. TypeScript autocomplete works when `@types/react` is installed.

## StringEventHandlers

Event handlers in Vincle are **strings**, not functions:

```tsx
<button onClick="console.log('clicked')">Click me</button>
```

This differs from React, where handlers are closures. The string is
HTML-escaped and inserted verbatim â€” suitable for simple interactions or
when paired with a client-side library (HTMX, Turbo, Alpine.js).

## When to use each type

| Type             | Use case                                                  |
| ---------------- | --------------------------------------------------------- |
| `VNode`     | `children` prop, return type of helpers, generic wrappers |
| `Component<P>`   | Annotating component functions                            |
| `HTMLAttributes` | Extending HTML elements in generic components             |
| `CSSProperties`  | Typing inline `style` objects                             |
