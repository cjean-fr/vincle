# Components

Components for deferring content in the HTML shell. Imported from `@vincle/flow`.

## Slot

Declares a named insertion point in the shell. Its children render immediately as placeholder content, visible until a `Template` replaces them.

<CodeExample src="flow/fill-signal.tsx" />

**Props:**

| Prop       | Type     | Default | Description                                                    |
| ---------- | -------- | ------- | -------------------------------------------------------------- |
| `name`     | `string` | —       | Fragment id, required. Must match `/^[a-zA-Z][a-zA-Z0-9_-]*$/` |
| `children` | `VNode`  | —       | Fallback content rendered immediately in the shell             |

## Template

Pushes content into a target slot by name. Unified — handles both sync and lazy content:

- **Sync** (plain JSX `children`) → registers content, no placeholder (was `Fill`)
- **Lazy** (factory `children`) → deferred, with optional `fallback` placeholder (was `Defer`)

<CodeExample src="flow/defer-signal.tsx" />

**Props:**

| Prop       | Type                                                        | Default     | Description                            |
| ---------- | ----------------------------------------------------------- | ----------- | -------------------------------------- |
| `target`   | `string`                                                    | —           | Target slot id to push content into    |
| `fallback` | `VNode`                                                     | —           | Placeholder shown while rendering lazy |
| `children` | `VNode \| ((signal: AbortSignal) => VNode)`                 | —           | Content (plain JSX or factory)         |
| `merge`    | `"replace" \| "append" \| "prepend" \| "before" \| "after"` | `"replace"` | How content applies to the target      |
| `timeout`  | `number`                                                    | —           | Per-fragment render timeout in ms      |
| `onError`  | `(error: Error) => VNode`                                   | —           | Per-fragment error handler             |

## Include

Browser fetches a fragment from a URL after the shell lands. No server-push required — works with any static file server.

## Content forms

`Template` accepts children in either form:

| Children                     | Behaviour                                                          |
| ---------------------------- | ------------------------------------------------------------------ |
| Plain JSX                    | Deferred automatically — no thunk needed                           |
| `(signal: AbortSignal) => …` | **Cancellable** — receives `AbortSignal` (request + timeout abort) |

Plain JSX works because `@vincle/flow` wraps children in a deferred execution
context: child JSX is captured at call time and its output is held back until
the fragment slot is rendered. A child like `<HeavyComponent />` is evaluated
inline by `@vincle/core`'s eager JSX factory, but its output is buffered and
emitted when the fragment is flushed.

Pass a factory when you need the `AbortSignal` for cancellation (request
disconnect, per-fragment timeout):

**Return types (for both forms):**

| Return type            | Behaviour                                            |
| ---------------------- | ---------------------------------------------------- |
| A renderable JSX node  | One-shot patch — renders once and resolves           |
| `Promise<VNode>`       | Resolves asynchronously, then patches in             |
| `AsyncIterable<VNode>` | A stream — one patch per item, as each item resolves |
